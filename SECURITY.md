# Security Policy — PathVision Final

This document outlines security practices, supported versions, and the reporting process for vulnerabilities in the PathVision Final project.

---

## 1. Supported Versions

Security updates are actively applied only to the latest release branches:

| Version | Supported |
| :--- | :--- |
| **v2.1.x** | :white_check_mark: Yes (Current Release) |
| **v2.0.x** | :x: No |
| **v1.x.x** | :x: No |

---

## 2. Reporting a Vulnerability

If you discover a security vulnerability in this project, please do not open a public issue. Instead, report it directly to the maintainers:

1. **Email**: Send a detailed report to `maintainer-email@example.com` (TODO: replace with release maintainer email).
2. **Details**: Include a description of the vulnerability, steps to reproduce, and any potential mitigations.
3. **Response**: We acknowledge receipt of reports within 48 hours and strive to resolve confirmed vulnerabilities within 14 days.
